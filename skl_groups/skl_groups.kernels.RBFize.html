skl_groups.kernels.RBFize¶

class skl_groups.kernels.RBFize(gamma=1, scale_by_median=False, squared=False, copy=True)¶

Turns a distance matrix into an RBF kernel: \(K(x, y) = \exp\left( - \gamma \lVert x - y \rVert^2 \right)\).

For the output to be a valid kernel, the inputs should either be Euclidean distances, e.g. the output of sklearn.metrics.pairwise.euclidean_distances(), or be isometrically embeddable into a Euclidean space. If not, you should ensure that the result is positive semidefinite before using it as a kernel, for example via ProjectPSD.

Parameters:

gamma : float, optional, default 1

The \(\gamma\) value to use in the kernel.

Defaults to 1, but this is not a very good value to use; two reasonable heuristics are 1 / feature dimension or 1 / the median distance value. You probably want to cross-validate on a range of possible values; see also scale_by_median.

scale_by_median : boolean, optional, default False

If True, actually use the median input distance times gamma.

squared : boolean, optional, default False

Whether the inputs are treated as distances or squared distances.

copy : boolean, optional, default True

If False, data passed to transform() is overwritten.

See also

sklearn.metrics.pairwise.rbf_kernel
computes this from feature vectors
